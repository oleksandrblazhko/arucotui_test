# Tasks

1) запропонувати протестувати 16-ть груп по 9-ть маркерів
2) послідовно розташовувати групи маркерів у чотирьох місцях розташування: 
- top - зверху від камери;
- center - у центрі перед камерою;
- left - зліва від камери
- rigth - зправа від камери із разворотом на 90 градусів вправо
- bottom - знизу від камери із разворотом на 180 градусів
3) для кожного місця розташування запропонувати розташування за різними кутами:
- кут = 0 градусів
- кут = 90 градусів за часовою стрілкою
- кут = 180 градусів за часовою стрілкою
- кут = 270 градусів за часовою стрілкою
4) записувати характеристики маркерів впродовж 10 секунд.
5) для кожного маркеру обчислити:
std(tvec_x), std(tvec_y), std(tvec_z), std(rvec_x), std(rvec_y), std(rvec_z)
6) для кожного маркеру обчислити:
- RSS-показник шуму (Root Sum Square of standard deviations) як евклідова норма вектора стандартних відхилень;
- detection_rate;
7) зберегти результати у файлі marker_test.json з елементами:
- location - місце розташування;
- angle - кут розташування;
- marker_id;
- rss
- detection_rate
8) провести аналіз, який був раніше у попередній версії програми



Для вашого проєкту важливіше не знайти "ідеальну камеру", а знайти набір ArUco ID, які максимально стійкі до реальних умов експлуатації:
зміни кута;
зміни положення в кадрі;
невеликого розмиття;
шуму камери;
помилок бінаризації.

Фактично ви займаєтесь тим, що в наукових роботах можна назвати:
Empirical robustness evaluation of ArUco markers under real imaging conditions.
