# Аналіз реорганізації архітектури ArUco сервера

## 1. Порівняння монолітної та модульної архітектур

### Монолітна архітектура (`server/marker.py`)

- Вся логіка зосереджена в одному файлі.
- Обробляє аргументи командного рядка (CLI), калібрування камери, налаштування камери, виявлення ArUco, оцінку пози, надсилання OSC-повідомлень та візуалізацію.
- Основний цикл перевантажений усіма цими завданнями.

### Модульна архітектура (`aruco_server/`)

- Логіка розділена на декілька файлів, кожен з яких має певну відповідальність:
  - `main.py`: Основна точка входу, яка керує іншими модулями.
  - `config.py`: Обробляє розбір аргументів командного рядка.
  - `calibration.py`: Завантажує дані калібрування камери.
  - `camera.py`: Керує налаштуванням та захопленням камери.
  - `aruco_detector.py`: Створює детектор ArUco та виявляє маркери.
  - `pose.py`: Оцінює пози маркерів.
  - `marker.py`: Створює об'єкт `MarkerData`.
  - `models/marker_data.py`: Визначає датаклас `MarkerData`.
  - `osc_sender.py`: Надсилає дані маркера через OSC.
  - `visualization.py`: Малює маркери та FPS на кадрі.
  - `filters/`: Новий додаток, цей каталог містить модулі для фільтрації/згладжування даних маркерів.

## 2. Втрачені властивості у модульній архітектурі

- **Шаблон словника ArUco:**
  - **Монолітна:** Дозволяє вибирати з `DICT_ARUCO_ORIGINAL`, `DICT_4X4_50` та `DICT_APRILTAG_36h11`. За замовчуванням використовується `DICT_4X4_50` (шаблон `1`).
  - **Модульна:** Дозволяє вибирати з `DICT_ARUCO_ORIGINAL`, `DICT_4X4_1000` та `DICT_APRILTAG_36h11`. Шаблон за замовчуванням також `1`, але він відповідає `DICT_4X4_1000` замість `DICT_4X4_50`. Це значна зміна. Опцію `DICT_4X4_50` було втрачено.

- **Візуалізація осей:**
  - **Монолітна:** Версія має цикл, який викликає `cv2.drawFrameAxes` для кожного виявленого маркера, малюючи осі X, Y та Z на відео.
  - **Модульна:** `visualization.py` модульної версії має лише `draw_markers` та `draw_fps`. Функціональність `drawFrameAxes` **відсутня**.

- **Нова функціональність (фільтрація):**
  - Модульна архітектура вводить значну нову функцію: згладжування/фільтрацію даних (`MarkerFilterManager`), якої не було в монолітній версії.

## 3. Аналіз помилок у модульній архітектурі

- **Помилки зв'язків (імпортів):**
  - У `aruco_server/main.py` є неправильний імпорт: `from marker_filter_manager import MarkerFilterManager`. Правильний шлях: `from filters.marker_filter_manager import MarkerFilterManager`.

- **Невідповідність словника:**
  - Зміна словника ArUco для шаблону `1` з `DICT_4X4_50` на `DICT_4X4_1000` є потенційною проблемою, яка може призвести до помилок у виявленні маркерів, якщо користувач очікує старого функціоналу.

- **Відсутність `drawFrameAxes`:**
  - Втрата візуалізації осей є втратою корисної функції для налагодження.
